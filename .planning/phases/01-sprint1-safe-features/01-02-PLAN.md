# Plan 01-02 — Auto-Cancel Stale Rides (Cron Job)

## Objective
Rides that no driver accepts within 2.5 minutes sit in `requested` status forever.
Add a backend cron that cancels them automatically and notifies the rider.

## Context
- @backend/src/routes/rides.ts — ride request, cancel, status transitions
- @backend/src/index.ts — app entry point (wire cron here)
- DB table: rides — columns: id, status, created_at, customer_id, driver_id
- Rider gets no notification today (push notifications are Sprint 2 — just set status for now)

## Tasks

### Task 1 — Install node-cron and write the stale-ride canceller
**Layer:** Backend
**Builds on:** supabaseAdmin client already available everywhere
**Action:**
1. `npm install node-cron @types/node-cron` in backend
2. Create `backend/src/jobs/cancelStaleRides.ts`:
   - SELECT rides WHERE status='requested' AND driver_id IS NULL AND created_at < NOW() - INTERVAL '2.5 minutes'
   - UPDATE those rides: status='cancelled', cancellation_reason='no_driver_available'
   - Log count of cancelled rides
3. In `backend/src/index.ts`, import and schedule: run every 60 seconds
**Done:** `cancelStaleRides.ts` exists; running it manually against DB cancels rides older than 2.5 min with no driver.

### Task 2 — Admin dashboard: show "no_driver_available" cancellation reason
**Layer:** Admin web (ActiveRides.tsx or SystemLogs.tsx)
**Builds on:** rides table already has status + cancellation data
**Action:**
- In ActiveRides.tsx, if a cancelled ride has reason `no_driver_available`, show a "Timed out" badge instead of just "Cancelled"
**Done:** Timed-out rides are visually distinct from rider-cancelled rides in the admin.
