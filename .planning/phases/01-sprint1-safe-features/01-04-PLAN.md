# Plan 01-04 — Admin: Ride Search + Filter

## Objective
Admin has no way to look up a specific ride. Add a searchable, filterable ride
history page so admins can find rides by status, date range, or driver/rider name.

## Context
- @backend/src/routes/admin.ts — has GET /riders/:id/rides but no global ride listing
- @admin-web/src/App.tsx — route definitions
- @admin-web/src/components/Layout.tsx — nav sidebar
- DB: rides table joins to profiles (customer), driver_profiles → profiles (driver)

## Tasks

### Task 1 — Add global ride listing endpoint
**Layer:** Backend (admin.ts)
**Builds on:** /riders/:id/rides already shows the query pattern; just remove the customer filter
**Action:**
Add `GET /api/v1/admin/rides?status=&from=&to=&page=&limit=20`:
- JOIN rides with customer profile (full_name) and driver profile (full_name)
- Filter by status if provided, date range if provided
- Return: id, status, pickup_address, dropoff_address, fare_estimate, final_fare,
  distance_km, payment_method, created_at, customer_name, driver_name
- Paginate (default limit 20)
**Done:** Endpoint returns paginated rides; filters work independently or combined.

### Task 2 — Build Rides history page in admin
**Layer:** Admin web
**Builds on:** Drivers.tsx and Riders.tsx show the card list pattern to follow
**Action:**
1. Create `admin-web/src/pages/Rides.tsx` with:
   - Filter bar: status dropdown, date-from/to pickers
   - Paginated ride cards (pickup → dropoff, driver, rider, fare, status badge)
   - Next/Prev pagination
2. Add route in App.tsx: `<Route path="rides/history" element={<Rides />} />`
3. Add "Ride History" link in Layout.tsx sidebar
**Done:** Admin can browse all rides, filter by status and date, navigate pages. No TS errors.
