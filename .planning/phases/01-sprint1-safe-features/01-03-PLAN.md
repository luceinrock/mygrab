# Plan 01-03 — Admin: Revenue & Ride Charts

## Objective
Replace the static stat cards on the Dashboard with time-series charts so the admin
can see ride volume and revenue trends over the last 7/30 days.

## Context
- @admin-web/src/pages/Dashboard.tsx — current stat cards, fetches /admin/analytics/overview
- @backend/src/routes/admin.ts — GET /analytics/overview (returns totals, not time-series)
- DB table: rides — columns: status, final_fare, fare_estimate, created_at

## Tasks

### Task 1 — Add time-series endpoint to backend
**Layer:** Backend (admin.ts)
**Builds on:** rides table has created_at and fare columns; overview endpoint already queries it
**Action:**
Add `GET /api/v1/admin/analytics/timeseries?days=7` (default 7, accept 7 or 30):
- Group rides by date (created_at::date)
- Return per-day: date, ride_count, completed_count, revenue (sum of final_fare for completed)
- Response: `{ series: [{ date: "2026-05-20", rides: 12, completed: 10, revenue: 1540.00 }] }`
**Done:** Endpoint returns correct per-day data; days=30 gives 30 rows max.

### Task 2 — Add Recharts to admin-web and build chart components
**Layer:** Admin web (Dashboard.tsx)
**Builds on:** Existing stat cards stay; charts go below them
**Action:**
1. `npm install recharts` in admin-web
2. Add a period toggle (7d / 30d) above the charts
3. Add two charts below the stat cards:
   - `<AreaChart>` — rides per day (total vs completed)
   - `<BarChart>` — revenue per day
4. Use Recharts' responsive container; keep styling consistent with existing Tailwind cards
**Done:** Dashboard shows two charts that update when toggling 7d/30d. No TypeScript errors.

## Verification
```bash
cd admin-web && npx tsc --noEmit
```
Visual: Charts render with real data; toggling period re-fetches and updates.
