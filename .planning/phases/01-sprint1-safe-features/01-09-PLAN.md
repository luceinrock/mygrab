# Plan 01-09 — Rider App: Fare Breakdown Before Confirming

## Objective
Riders currently request a ride without seeing what it will cost.
Show a fare estimate sheet after setting pickup/dropoff, before confirming.

## Context
- @android-rider/app/src/main/java/com/ridenow/rider/ui/home/HomeScreen.kt
- @android-rider/app/src/main/java/com/ridenow/rider/ui/home/HomeViewModel.kt
- Backend: GET /api/v1/pricing/estimate?pickup_lat=&pickup_lng=&dropoff_lat=&dropoff_lng=&vehicle_type=
  Returns: fare_estimate, distance_km, duration_min, base_fare, surge_multiplier

## Tasks

### Task 1 — Fetch estimate in HomeViewModel before requesting
**Layer:** Android rider app
**Builds on:** HomeViewModel already has pickup/dropoff coordinates; API client pattern established
**Action:**
1. Add `@GET("api/v1/pricing/estimate") suspend fun getEstimate(@QueryMap params: Map<String, String>): Response<EstimateResponse>` to RideNowApi
2. Add `EstimateResponse` data class: fare_estimate, distance_km, duration_min, base_fare, surge_multiplier
3. In HomeViewModel, add `fetchEstimate()` that calls the endpoint and stores result in UiState
4. Change the "Request Ride" button flow: first call fetchEstimate() → show bottom sheet → user confirms → then call requestRide()
**Done:** Estimate fetched without error; UiState contains estimate data before ride is created.

### Task 2 — Build fare breakdown bottom sheet
**Layer:** Android rider app
**Builds on:** Jetpack Compose ModalBottomSheet available in the dependency set
**Action:**
1. Add `ModalBottomSheet` that shows when estimate is ready:
   - Distance (e.g. "3.2 km")
   - Duration (e.g. "~8 min")
   - Fare breakdown rows: Base fare, Distance charge, Surge (only if >1x), Booking fee, **Total**
   - Vehicle type selector (Lite / Plus / Moto) that re-fetches estimate on change
   - "Confirm Ride" and "Cancel" buttons
2. Tapping Confirm → calls requestRide() → dismisses sheet
**Done:** Bottom sheet renders with correct values; vehicle type change updates total; Confirm triggers ride request.
