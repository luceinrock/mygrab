# Plan 01-10 — Rider App: Saved Favorite Addresses

## Objective
Riders frequently go to the same places (home, work). Let them save two favorites
(Home, Work) so they can tap instead of typing every time.

## Context
- @android-rider/app/src/main/java/com/ridenow/rider/ui/home/HomeScreen.kt
- @android-rider/app/src/main/java/com/ridenow/rider/ui/home/HomeViewModel.kt
- Backend: GET/PUT /api/v1/riders/profile — profile has saved_pickup_lat/lng, saved_dropoff_lat/lng already?
- DB: customer_profiles table — check what columns exist

## Tasks

### Task 1 — Add favorites columns to customer_profiles if missing
**Layer:** Supabase migration + backend
**Builds on:** /api/v1/riders/profile PUT already accepts arbitrary profile fields
**Action:**
1. Check customer_profiles schema; if home/work columns missing, apply migration:
   ```sql
   ALTER TABLE customer_profiles
     ADD COLUMN IF NOT EXISTS home_address TEXT,
     ADD COLUMN IF NOT EXISTS home_lat NUMERIC,
     ADD COLUMN IF NOT EXISTS home_lng NUMERIC,
     ADD COLUMN IF NOT EXISTS work_address TEXT,
     ADD COLUMN IF NOT EXISTS work_lat NUMERIC,
     ADD COLUMN IF NOT EXISTS work_lng NUMERIC;
   ```
2. Update GET /api/v1/riders/profile to include these fields in SELECT
3. Update PUT /api/v1/riders/profile Zod schema to accept them
**Done:** GET /riders/profile returns home/work fields; PUT saves them.

### Task 2 — Add favorites UI to rider home screen
**Layer:** Android rider app
**Builds on:** HomeScreen already has pickup/dropoff input; HomeViewModel has address state
**Action:**
1. Load home/work from profile on HomeViewModel init
2. Show two chip buttons below the address inputs: 🏠 Home  💼 Work (only visible if that favorite is saved)
3. Tapping a chip fills in dropoff address + coordinates
4. In a simple Settings or Profile screen, let rider save current dropoff as Home or Work
**Done:** Home/Work chips appear when saved; tapping pre-fills dropoff; chips hidden when not yet saved.
