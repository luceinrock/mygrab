# Plan 01-08 — Rider App: Cancel Ride Button

## Objective
Riders can request a ride but have no way to cancel it from the app if they change
their mind before a driver arrives. Add a cancel button with a confirmation dialog.

## Context
- @android-rider/app/src/main/java/com/ridenow/rider/ui/ride/ActiveRideScreen.kt
- @android-rider/app/src/main/java/com/ridenow/rider/ui/ride/RideViewModel.kt
- Backend: POST /api/v1/rides/:id/cancel — already implemented, sets status=cancelled,
  increments cancellation_strikes if driver already accepted

## Tasks

### Task 1 — Add cancelRide() to RideViewModel and RiderApi
**Layer:** Android rider app
**Builds on:** RideViewModel already manages active ride state; API client already has pattern
**Action:**
1. Add `@POST("api/v1/rides/{id}/cancel") suspend fun cancelRide(@Path("id") id: String): Response<Unit>` to the rider RideNowApi interface
2. In RideViewModel, add:
   ```kotlin
   fun cancelRide() {
       viewModelScope.launch {
           _uiState.update { it.copy(isCancelling = true) }
           try {
               api.cancelRide(activeRide.id)
               _uiState.update { it.copy(activeRide = null, isCancelling = false) }
           } catch (e: Exception) {
               _uiState.update { it.copy(isCancelling = false, error = "Could not cancel. Try again.") }
           }
       }
   }
   ```
3. Add `isCancelling: Boolean` and `showCancelDialog: Boolean` to the UiState
**Done:** cancelRide() compiles and correctly clears active ride on success.

### Task 2 — Add Cancel button + confirmation dialog to ActiveRideScreen
**Layer:** Android rider app
**Builds on:** ActiveRideScreen already shows status, pickup/dropoff addresses
**Action:**
1. Show "Cancel Ride" button only when status is `requested` or `accepted` (hide during in_progress)
2. Tapping opens an AlertDialog: "Cancel this ride? If a driver already accepted, this may affect your account."
3. Confirm → calls viewModel.cancelRide(); Dismiss → closes dialog
4. Show a loading indicator while isCancelling = true; disable button
**Done:** Cancel button visible in correct states; dialog shows; confirmed cancel navigates back to home.
