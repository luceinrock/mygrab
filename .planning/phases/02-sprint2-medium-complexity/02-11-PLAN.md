# Plan 02-11 — Push Notifications via FCM

## Objective
Drivers currently poll for rides every 3 seconds. Riders have no notification when
a driver is assigned. Replace polling with FCM push so both apps wake up on events.

## Context
- @android-driver/app — uses polling in HomeViewModel
- @android-rider/app — no ride assignment notification today
- @backend/src/routes/rides.ts — accept, complete, cancel endpoints
- @backend/src/routes/drivers.ts — toggle-online
- Firebase project must be created (or connected to existing)

## Pre-work (human action required)
`checkpoint:human-action` — Create a Firebase project at console.firebase.google.com,
enable Cloud Messaging, download `google-services.json` for both apps.
Drop `android-driver/app/google-services.json` and `android-rider/app/google-services.json`.

## Tasks

### Task 1 — Backend: FCM integration service
**Layer:** Backend
**Action:**
1. `npm install firebase-admin` in backend
2. Add FIREBASE_SERVICE_ACCOUNT_JSON to Render env vars (base64 encoded)
3. Create `backend/src/services/notificationService.ts`:
   - `sendToToken(token, title, body, data)` — wraps FCM sendEachForMulticast
   - `notifyDriver(driverId, payload)` — looks up driver's fcm_token from DB, sends
   - `notifyRider(riderId, payload)` — same for rider
4. Add `fcm_token TEXT` column to both `driver_profiles` and `customer_profiles` via migration
5. Add `POST /api/v1/drivers/fcm-token` and `POST /api/v1/riders/fcm-token` endpoints to store token on login
6. Wire notifications into rides.ts:
   - New ride requested → notify all online drivers (or nearest N after phase 12)
   - Ride accepted → notify rider ("Driver is on the way")
   - Driver arrived → notify rider ("Your driver has arrived")
   - Ride completed → notify both
   - Ride cancelled → notify the other party
**Done:** Backend sends FCM messages; verified via Firebase console delivery reports.

### Task 2 — Android: FCM setup + token registration
**Layer:** Both Android apps
**Builds on:** google-services.json added in pre-work
**Action:**
1. Add to both apps' build.gradle: `com.google.firebase:firebase-messaging`
2. Create `FirebaseMessagingService` in each app that:
   - On `onNewToken`: POST token to backend (/drivers/fcm-token or /riders/fcm-token)
   - On `onMessageReceived`: show notification if app is in background; handle foreground
3. Driver app: remove 3-second polling for available rides (HomeViewModel) — rely on FCM to wake up, then fetch
4. Show a local notification for incoming rides; tapping opens the app to the ride detail
**Done:** Driver receives push when a ride is requested; polling loop removed; rider notified on driver assignment.

## Verification
End-to-end: Request a test ride → driver app receives push notification within 5 seconds → accept → rider app gets "Driver on the way" notification.

## Checkpoint
`checkpoint:human-verify` — Confirm notifications arrive on physical devices (not just emulator) for all events.
