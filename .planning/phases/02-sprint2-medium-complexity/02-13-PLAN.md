# Plan 02-13 — Promo Codes / Discounts

## Objective
Allow riders to enter a promo code at booking time to get a discount on their fare.
Admin can create, manage, and deactivate promo codes from the dashboard.

## Context
- @backend/src/routes/rides.ts — POST /rides/request calculates fare
- @backend/src/routes/admin.ts — admin management routes
- @admin-web/src/pages/ — admin UI
- @android-rider/app — fare breakdown bottom sheet (from phase 01-09)

## Tasks

### Task 1 — Promo codes DB schema + backend CRUD
**Layer:** Supabase migration + Backend (admin.ts)
**Action:**
1. Migration:
   ```sql
   CREATE TABLE promo_codes (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     code TEXT UNIQUE NOT NULL,
     description TEXT,
     discount_type TEXT NOT NULL CHECK (discount_type IN ('percent', 'fixed')),
     discount_value NUMERIC(8,2) NOT NULL,
     max_uses INTEGER,
     uses_count INTEGER NOT NULL DEFAULT 0,
     min_fare NUMERIC(8,2) DEFAULT 0,
     expires_at TIMESTAMPTZ,
     is_active BOOLEAN NOT NULL DEFAULT true,
     created_at TIMESTAMPTZ DEFAULT NOW()
   );
   ```
2. Admin endpoints in admin.ts:
   - `GET /api/v1/admin/promo-codes` — list all
   - `POST /api/v1/admin/promo-codes` — create
   - `PATCH /api/v1/admin/promo-codes/:id/deactivate` — deactivate
**Done:** Admin can create a code "GRAB50" for 50% off; code appears in list; deactivate sets is_active=false.

### Task 2 — Validate and apply promo at ride request
**Layer:** Backend (rides.ts)
**Builds on:** POST /rides/request already calculates fare_estimate
**Action:**
1. Add `POST /api/v1/riders/validate-promo` endpoint:
   - Input: code, fare_amount
   - Check: is_active, not expired, uses_count < max_uses (if set), fare >= min_fare
   - Return: { valid: true, discount_amount, final_fare } or { valid: false, reason }
2. In POST /rides/request: accept optional promo_code; validate it; apply discount to fare; increment uses_count; store promo_code on rides row
3. Add `promo_code TEXT, discount_amount NUMERIC` columns to rides table
**Done:** Applying valid code reduces fare; used code increments counter; expired/maxed codes rejected.

### Task 3 — Promo code input in rider app + admin management UI
**Layer:** Android rider app (fare bottom sheet from 01-09) + Admin web
**Action:**
1. Rider app: Add a "Have a promo code?" text field in the fare breakdown sheet
   - On apply: call validate-promo; show discount line in breakdown; update total
2. Admin web: Add "Promo Codes" page (list + create form + deactivate button)
**Done:** Rider enters code → sees discount applied in fare sheet → confirmed ride uses discounted fare. Admin can see usage count per code.
