# Plan 02-12 — Driver Proximity Matching

## Objective
When a ride is requested, ALL online drivers are notified (or polled). This means
a driver 30km away sees the same ride as one 500m away. Implement proximity-based
matching: only notify the N nearest available drivers (e.g. nearest 5 within 10km).

## Context
- @backend/src/routes/rides.ts — POST /rides/request creates ride and notifies
- @backend/src/services/matching.ts — matching service (check what's already there)
- DB: driver_profiles has current_location_lat, current_location_lng
- Phase 02-11 (FCM) should be complete — this replaces broadcast with targeted notify

## Tasks

### Task 1 — Haversine proximity query in matching service
**Layer:** Backend (matching.ts)
**Builds on:** driver_profiles already has lat/lng; matching.ts exists
**Action:**
1. Add `getNearestDrivers(pickupLat, pickupLng, radiusKm=10, limit=5)`:
   - Use PostgreSQL's earth_distance extension OR inline Haversine formula in SQL:
     ```sql
     SELECT user_id,
       (6371 * acos(cos(radians($1)) * cos(radians(current_location_lat)) *
        cos(radians(current_location_lng) - radians($2)) +
        sin(radians($1)) * sin(radians(current_location_lat)))) AS distance_km
     FROM driver_profiles
     WHERE is_online = true AND is_available = true AND verification_status = 'verified'
       AND last_location_update > NOW() - INTERVAL '5 minutes'
     HAVING distance_km < $3
     ORDER BY distance_km ASC
     LIMIT $4
     ```
2. Add index on driver_profiles(is_online, is_available) for query performance
**Done:** getNearestDrivers returns correct drivers ordered by distance; tested with known coordinates.

### Task 2 — Wire proximity into ride request + FCM notify
**Layer:** Backend (rides.ts)
**Builds on:** Phase 02-11 FCM notificationService already built
**Action:**
1. In POST /rides/request: after creating the ride, call getNearestDrivers(pickup_lat, pickup_lng)
2. If drivers found: notify each via FCM (notifyDriver) with ride details
3. If no drivers within 10km: notify all online drivers as fallback (broadcast)
4. Available-rides polling endpoint still works as fallback for drivers without FCM
**Done:** New ride notifies only nearby drivers; driver 20km away does NOT receive notification for a local ride.

## Verification
Manual test: Two drivers online — one near pickup (1km), one far (15km). Request ride.
Only the near driver receives FCM push. Far driver's available-rides poll still shows the ride (fallback).
